(EN - US) Notes:

* Final" word in the class: Prevents that class from being inherited;

* Final word in the method: Prevents the method from being overridden.

* Final word in the variable: Prevents the variable value from being modified.

* It is usually best to add "final" to overridden methods, because multiple overlaps can be a gateway to inconsistencies.

* Type attributes of a "final" class are parsed faster at runtime.

(PT - BR) Observações:

* Palavra "final" na classe: Evita que aquela classe seja herdada;

* Palavra "final" no método: Evita que o método seja sobreposto.

* Palavra "final" na variável: Evita que o valor da variável seja modificado.

* Geralmente convém acrescentar "final" em métodos sobrepostos, pois sobreposições múltiplas podem ser uma porta de entrada para inconsistências.

* Atributos de tipo de uma classe "final" são analisados de forma mais rápida em tempo de execução.
